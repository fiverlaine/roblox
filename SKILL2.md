# Frontend Design — Knowledge Base

> **Purpose:** This knowledge defines the required design philosophy and frontend standards for every UI component, page, or interface built in this project. The AI must consult and follow these rules on every code generation involving visual output.

---

## 1. Core Design Philosophy

Every interface must feel **deliberately designed** — not generic, not "AI-generated." Before writing any UI code, commit to a clear aesthetic direction and execute it with full intentionality.

Ask these questions first:
- **What problem does this interface solve? Who is the user?**
- **What tone fits this context?** Choose one extreme and own it: brutally minimal, maximalist chaos, retro-futuristic, organic/natural, luxury/refined, playful/toy-like, editorial/magazine, brutalist/raw, art deco/geometric, soft/pastel, industrial/utilitarian.
- **What makes this interface UNFORGETTABLE?** Define the single most memorable aspect and build around it.

---

## 2. Typography Rules

- **NEVER** use generic fonts: `Inter`, `Roboto`, `Arial`, `system-ui`, or `Space Grotesk`.
- **ALWAYS** pair a distinctive display font with a refined body font.
- Font choices must feel unexpected and intentional — chosen specifically for the emotional context of the interface.
- Typography is the primary differentiator. Invest in it.

---

## 3. Color & Theme Rules

- Commit to a **cohesive visual identity** using CSS custom properties (`--color-*` variables).
- Use a **dominant color + sharp accent** strategy. Avoid timid, evenly-distributed neutral palettes.
- Alternate between light and dark themes across different interfaces — do not always default to the same mode.
- **NEVER** use clichéd color combinations (e.g., purple gradients on white backgrounds).

---

## 4. Motion & Animation

- Animations must be purposeful, not decorative filler.
- Prioritize **one high-impact, orchestrated moment**: a staggered page-load reveal using `animation-delay` is more effective than scattered micro-interactions.
- Use CSS-only animations whenever possible for HTML. Use the **Motion** library (`motion/react`) for React components.
- Add hover states and scroll-triggered effects that feel surprising and delightful.

---

## 5. Layout & Spatial Composition

- Use **unexpected layouts**: asymmetry, overlapping elements, diagonal flow, grid-breaking components.
- Generous negative space OR controlled density — choose one and apply it consistently.
- Avoid predictable, grid-only, cookie-cutter arrangements.

---

## 6. Backgrounds & Visual Depth

Do not default to solid flat colors. Create **atmosphere**:
- Gradient meshes, noise textures, geometric patterns
- Layered transparencies and dramatic box shadows
- Decorative borders, custom cursors, grain overlays
- Contextual effects that match the chosen aesthetic

---

## 7. Implementation Quality

- All code must be **production-grade and fully functional** — no placeholder logic, no incomplete features.
- Match implementation complexity to the aesthetic vision:
  - **Maximalist designs** → elaborate code, extensive animations, rich effects.
  - **Minimalist designs** → restraint, precise spacing, careful typography, subtle details.
- Every visual detail must be intentional and refined.

---

## 8. Absolute Prohibitions (Never Do)

- ❌ Never use overused font families: `Inter`, `Roboto`, `Arial`, `system-ui`, `Space Grotesk`
- ❌ Never use purple gradients on white backgrounds as a default aesthetic
- ❌ Never produce predictable, cookie-cutter layouts
- ❌ Never output two interfaces with the same aesthetic direction
- ❌ Never use inline styles for theming — always use CSS variables or a design token system
- ❌ Never converge on the same aesthetic across multiple generations

---

## 9. Mandatory Creative Checklist

Before finalizing any UI output, verify:

- [ ] A clear, named aesthetic direction was chosen (e.g., "editorial brutalism", "organic luxury")
- [ ] Font pairing is distinctive and context-appropriate
- [ ] Color palette uses CSS variables with dominant + accent strategy
- [ ] At least one high-impact animation or motion moment exists
- [ ] Layout has at least one unexpected spatial decision
- [ ] Background has depth (not a plain solid color)
- [ ] The design would be immediately recognizable as intentionally crafted — not AI-generated slop

---

## 10. Summary Directive

> **The AI must produce interfaces that look and feel like they were designed by a senior product designer with a strong point of view — not generated by a default AI prompt. Every output must be visually striking, cohesive, and memorable. Generic is failure.**
